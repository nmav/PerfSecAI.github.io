Protect your infrastructure from malware threats

The Python Package Index (PyPI), is a vital repository for myriad Python software projects. Due to its popularity it has been weaponized by malicious actors deploying malware through typosquatting—subtly misspelling package names to trap unwary developers into incorporating hazardous code. This tactic can devastate your organizational security exposing the organization to unprecedented risk.

Harden the security of your development teams, ensuring that only expert-vetted and digitally signed packages are interwoven into your codebase. Automate SBOM Generation

Providing a Software Bill of Materials (SBOM) has become a non-negotiable standard in software delivery, offering a transparent lens through which end users can identify and navigate potential vulnerabilities, especially during critical cybersecurity crises like the notorious log4j incident. The SBOM not only signifies a commitment to transparency but also fortifies your user's trust, allowing them to identify and mitigate vulnerable components swiftly and effectively during pressing situations, thereby safeguarding their operations and maintaining the integrity of their digital assets.

Install and use perfsec in 3 easy steps

1. Get your token

You first need a token for your instances to use perfsec. Visit perfsec.ai and get to the Token Management page. Click refresh and copy the text from the token field.

2. Set the Python package provider

Type the following command into the terminal.

pip config set --site global.index-url https://<token>@index-staging.perfsec.ai/simple/

3. Install the wyse tooling or any other package

Type the following commands into the terminal.

pip install wyse
pip install <package>
